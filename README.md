# 💰 Skypro Wallet - Учет личных расходов

Приложение для ведения личных финансов: добавление, удаление, анализ расходов по категориям и периодам.


## ✨ Функциональность

- ✅ Регистрация и вход (локальное хранилище `localStorage`)
- ✅ Создание, просмотр, удаление расходов
- ✅ Категории: Еда, Транспорт, Жилье, Развлечения, Образование, Другое
- ✅ Аналитика расходов за выбранный период (календарь с выбором диапазона)
- ✅ Визуализация: столбчатая диаграмма расходов по категориям
- ✅ Интерактивные формы с валидацией
- ✅ **Интеграция с реальным API** (`https://wedev-api.sky.pro/api`) для хранения расходов

## 🧰 Технологии

- **React** (функциональные компоненты, хуки)
- **React Router DOM** – маршрутизация
- **Styled Components** + **CSS Modules** – стилизация
- **React Icons** – иконки
- **Fetch API** – HTTP-запросы к бэкенду
- **date-fns** – работа с датами

🔐 Авторизация
Особенность: для удобства разработки авторизация реализована через localStorage (эмуляция). Пользователи создаются локально и не отправляются на сервер.

Расходы при этом сохраняются на реальном бэкенде. Для работы с API используется фиксированный Bearer-токен, который вручную вставляется в код (файл src/api/transactionsApi.js). Это позволяет тестировать функциональность без полноценной настройки OAuth.

⚠️ При первом использовании необходимо получить токен через Postman или curl, отправив запрос на https://wedev-api.sky.pro/api/auth/login с тестовыми учётными данными (предоставленными преподавателем). Полученный токен вставить в переменную FIXED_TOKEN в файле src/api/transactionsApi.js.

📡 API Расходов
Приложение использует следующие эндпоинты:

Метод	URL	Описание
GET	/api/transactions	Получить все расходы
POST	/api/transactions	Добавить новый расход
DELETE	/api/transactions/:id	Удалить расход
POST	/api/transactions/period	Получить расходы за период
Формат данных:

_id – идентификатор транзакции

description – описание

category – категория (food, transport, housing, joy, education, others)

date – дата в формате YYYY-MM-DD или MM-DD-YYYY (адаптер преобразует)

sum – сумма (в API называется sum, в приложении – amount)


Расходы не загружаются – проверьте, что в transactionsApi.js вставлен актуальный токен.
